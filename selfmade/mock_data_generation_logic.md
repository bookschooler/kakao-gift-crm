## 3. 구매 빈도 분포 (Purchase Frequency)

### 왜 Poisson 분포인가?
선물하기는 **사회적 이벤트(생일, 명절, 기념일)에 종속된 구매**. 이런 구조는:
- 각 이벤트는 서로 독립적으로 발생
- 단위 시간당 평균 발생률이 일정 (연 1~4회)
- 중복 발생 없음 (같은 날 같은 사람에게 두 번 보내지 않음)

→ **Poisson 분포 (λ=4/year)** 가 적합

```
Gemini 검증 포인트: "카카오톡이라는 사회적 네트워크 내 구매는
개인 소비재의 Pareto 분포와 달리 사회적 제약(이벤트 수)이
구매 횟수의 상한을 제한함. Poisson이 더 현실적."
```

### 실제 데이터 기반 λ 설정
- 카카오 공식: MAU 2,600만 / 일 거래 60만건 = 1인당 월 약 0.69건 → **연 약 8.3건**
- 그러나 이는 MAU(월 활성 유저) 기준. 2년 누적 유저는 비활성 포함.
- **비활성 유저(1년 이상 미구매) 30% 가정** → 활성 유저 연 11.8건, 전체 평균 λ=4~6 사이
- **최종 선택: λ=4** (2년 기간, 시즌 부스트 별도 반영)

## 5. 상품 가격 분포 (Price Distribution)

### 왜 Log-normal 분포인가?
선물 가격은:
- 하한 있음 (최소 1,000원 이상)
- 오른쪽 꼬리 길음 (명절 선물세트 10~20만원)
- 대다수가 10,000~30,000원대 집중

→ **Log-normal(μ=10, σ=0.6)** 적합 (10,000~30,000원 모드)

### 실제 데이터 기반 가격대
| 카테고리 | 주요 가격대 | 대표 상품 | 적용 분포 |

| food (snack류) | 5,000~15,000원 | 빼빼로 세트 | Mid Log-normal |
| beauty | 15,000~50,000원 | 화장품·향수 | High Log-normal |
| health (홍삼) | 30,000~150,000원 | 홍삼 세트, 건기식 | Very High Log-normal |
| voucher (백화점) | 10,000~50,000원 | 문화상품권 | Mid Log-normal |

### 실제 데이터 출처
| 채널 | 지표 | 수치 | 출처 |
|---|---|---|---|

| 카카오 메시지 전반 | 차단율 | 0.5~2% | 오픈애즈(빅인) CRM 전략 가이드 [⑦] |

### A/B 테스트 설계 근거
```
Curation(A안) vs Ranking(B안) 메시지 전략:

A안 - Curation (개인화 추천):
  "Sophie님이 좋아할 것 같은 향수 TOP3"
  → CTR: 6% (낮음 — 개인화지만 니즈가 낮을 수 있음)
  → CVR: 15% (높음 — 클릭하면 관심 있는 상품)
  → Block rate: 0.8%

B안 - Ranking (베스트셀러):
  "이번 주 가장 많이 선물된 TOP10"
  → CTR: 18% (높음 — 사회적 증거 효과)
  → CVR: 4% (낮음 — 클릭했지만 내 취향 아닐 수 있음)
  → Block rate: 1.5% (높음 — 일반적 메시지 피로도)

설계 의도:
- CTR은 B가 높지만 CVR은 A가 높음 → 퍼널 전체 효율은 어느 쪽?
- 이것이 Phase 5 A/B 분석의 핵심 질문
```

### 채널별 특성 (2025년 카카오 2체계 개편 반영)
```
카카오 메시지 2체계:
  알림톡 → 정보성 (주문 확인, 배송 알림) — 캠페인 대상 아님
  브랜드 메시지 → 광고성 (기존 친구톡 흡수, 마케팅 수신 동의자 전체 발송 가능)

campaigns.channel 적용:

  in_app_message: 앱 내 팝업/배너 (선물하기 탭 진입 시)
```
