## 전체 진단: 왜 지금 피치가 약한가

| 문제 유형 | 현상 | 영향 |
|-----------|------|------|
| **프레임 문제** | "내가 무엇을 분석했다" 중심 서술 | 청중이 자신의 문제로 느끼지 못함 |
| **강조 부재** | 모든 수치가 동등하게 나열 | 핵심 인사이트가 묻혀버림 |
| **의사결정 가이드 없음** | 분석 결과만 있고 "언제 써야 하나"가 없음 | 실행 연결이 끊어짐 |
| **약점 노출 프레임** | "DESA 추정값 수정" 등 표현 | 분석 신뢰도 스스로 깎음 |
| **근거 없는 목표 수치** | "데이터 품질 92%" 등 달성 목표 | 합성 데이터 기반에서 신뢰도 역효과 |

## 구조 개선: 흐름 리프레임

### 기존 흐름 (문제)
```
표지 → 문제 정의 → 분석 설계 → Phase 1→5 나열 → 요약 → 한계
```
→ "분석자가 뭘 했나" 관점

### 개선 흐름
```
표지 (임팩트 있는 부제)
↓
선물하기 팀이 지금 직면한 현실 (슬라이드 2 리프레임)
↓
분석 설계 (Why this structure — 비즈니스 질문 중심)
↓
5 Phase 분석 (각 슬라이드 = 질문 → 데이터 → 답)
↓
핵심 요약 + 실행 제언 크게
↓
로드맵 (비즈니스 임팩트 포함)
↓
한계 (약점 방어, 미래 보완 프레임)
```
→ "카카오 팀 문제에 답하는 사람" 관점

## 슬라이드별 수정 계획

### 슬라이드 1 — 표지

| | 기존 | 수정 |
|--|------|------|
| **부제** | "성장 지속을 위한 사용자 행동 기반 전략 제언" | **"GMV +29.7% 성장 이면, 우리가 놓치고 있는 3가지"** |

**수정 이유:** 기존 부제는 전형적인 포트폴리오 문구. 수정 버전은 청중에게 즉시 "내 이야기"로 느끼게 하는 훅(hook). 숫자를 앞에 배치해 신뢰도 선제 확보.

### 슬라이드 2 — 문제 정의

| 위치 | 기존 | 수정 |
|------|------|------|
| **상단 배너** | "카카오톡 선물하기 현황" | **"선물하기 팀이 지금 직면한 현실"** |
| **카드 1** | "GMV 성장이 이벤트 시즌에만 집중된 건 아닌가?" | **"설날 3일이 월 GMV의 40%를 만든다. 비시즌 마케팅 예산은 어떻게 정당화하는가?"** |
| **카드 2** | "어떤 사용자가 실제 비즈니스를 떠받치는가?" | **"Dormant 44.5% — 유저 절반이 자고 있다. CRM 예산을 어디에 집중해야 하는가?"** |
| **카드 3** | "수신자가 발신자로 전환되는 구조가 얼마나 효율적인가?" | **"수신자 97.9%가 발신자가 된다. 그런데 왜 63.2%는 1년이 넘어도 전환하지 않는가?"** |

**수정 이유:** 기존 카드는 교과서적 질문 형식. 수정 버전은 구체적 수치를 포함한 "긴장감 있는 현실 진술" 형식. 카카오 실무자가 실제로 고민하는 형태로 재구성.

### 슬라이드 3 — 분석 설계

| | 기존 | 수정 |
|--|------|------|
| **Phase 박스** | 방법론 이름만 있음 | 각 Phase 아래 비즈니스 질문 1줄 추가 |

**추가할 비즈니스 질문:**

| Phase | 기존 | 추가 질문 |
|-------|------|----------|
| 1 EDA | 시즌 의존도 진단 | "비시즌 예산을 써도 되는가?" |
| 2 RFM | 고가치 고객 식별 | "어디에 CRM 예산을 집중하는가?" |
| 3 LTV | 고객 장기 가치 산출 | "신규 고객 1명에 얼마까지 써도 되는가?" |
| 4 Viral | K-factor 분석 | "바이럴이 실제로 작동하고 있는가?" |
| 5 CRM | A/B·ROAS·액션 | "어떤 메시지를 언제 보내야 하는가?" |

**수정 이유:** 방법론 나열은 "내가 뭘 했다" 자랑. 비즈니스 질문 병기는 "이걸 왜 했는지"를 즉시 전달.

```
언제 Ranking?               언제 Curation?
─────────────────          ─────────────────
```

**수정 이유:** A/B 결과만 있고 "언제 어떤 걸 써야 하나"가 없으면 실무자 입장에서 쓸모가 없음. 의사결정 가이드가 분석을 실행으로 연결하는 다리.

**수정 이유:** 카카오 팀이 가장 보고 싶은 건 분석 결과가 아니라 "우리가 내일 당장 뭘 해야 하는가". 실행 제언이 가장 크게 보여야 함.
